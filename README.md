# ניהול ועד — הוראות הקמה

אפליקציה לניהול ועד בית: הכנסות, הוצאות, מעקב דירות ודיירים, קבלות עם חתימה ושיתוף
בוואטסאפ, דוחות תקופתיים, ומעקב פרויקטים/הצעות מחיר.

## 1. הקמת פרויקט Firebase (בדיוק כמו ב-eWay)

1. כנס ל-https://console.firebase.google.com ופתח פרויקט חדש.
2. **Firestore Database** → צור מסד נתונים (Production mode).
3. **Authentication** → Sign-in method → הפעל **Email/Password**.
4. **Authentication** → Users → הוסף משתמש אחד עבורך (זה יהיה חשבון הוועד).
5. **Project settings** → Your apps → הוסף אפליקציית Web → העתק את פרטי ה-config.
6. הדבק את הפרטים בקובץ `firebase-config.js` (מחליף את `REPLACE_ME`).

## 2. חוקי אבטחה ב-Firestore (Firestore Rules)

הדבק בלשונית Rules:

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    function isAdmin() {
      return request.auth != null;
    }
    match /settings/{doc} {
      allow read: if true;
      allow write: if isAdmin();
    }
    match /units/{doc} {
      allow read: if true;
      allow write: if isAdmin();
    }
    match /transactions/{doc} {
      allow read: if true;
      allow write: if isAdmin();
    }
    match /projects/{doc} {
      allow read: if true;
      allow write: if isAdmin();
    }
  }
}
```

**חשוב להבין**: רק אתה (המחובר עם אימייל/סיסמה) יכול לכתוב/לשנות נתונים —
זה השומר העיקרי. הקריאה פתוחה טכנית לכל מי שניגש ישירות ל-Firestore (לא
דרך האפליקציה), כדי לאפשר את מסך הצפייה הציבורי בלי התחברות מורכבת. קוד
ה-PIN של דיירים נבדק בצד הלקוח (בדפדפן) ולא ברמת השרת. עבור אפליקציית ועד
בית של 12 דיירים זה סביר, אבל זה לא רמת אבטחה בנקאית — אם בעתיד תרצה הקשחה
נוספת (למשל דרך Cloud Functions), אפשר לשדרג.

## 3. פריסה ל-GitHub Pages

זהה בדיוק לתהליך שעשית עם eWay:
1. צור repository חדש ב-GitHub (למשל `nihul-vaad`).
2. העלה את כל הקבצים בתיקייה הזו (`index.html`, `app.js`, `styles.css`, `firebase-config.js`).
3. Settings → Pages → הפעל מ-branch `main`, תיקיית root.
4. האתר יהיה זמין תוך דקה בכתובת `https://USERNAME.github.io/nihul-vaad/`.

## 4. הגדרה ראשונית באפליקציה

1. היכנס עם חשבון הוועד (אימייל/סיסמה).
2. עבור למסך **הגדרות** → הזן שם ועד, כתובת הבניין, דמי ועד חודשיים, וצייר חתימה.
3. לכל דירה לחץ "ערוך" והזן שם דייר נוכחי, טלפון (לשיתוף קבלות), וקוד PIN
   (הדייר ישתמש בו כדי להתחבר למסך האישי שלו).

## 5. ייבוא הנתונים ההיסטוריים (2023–2026) מהקובץ המקורי

בקובץ `historicalData.js` נמצאים כל התנועות שחולצו מהאקסל המקורי שלך —
506 רשומות הכנסה ו-129 רשומות הוצאה, כולל פרויקט ה"בלת"ם" (פיצוץ צנרת
2024) והצעות המחיר לשיפוץ חדר המדרגות. הסכומים הכוללים נבדקו מול תיבות
הסיכום שהיו בכל גיליון שנה בקובץ המקורי והם תואמים במדויק.

**גיליון "2022" לא יובא** — התברר בבדיקה שהוא זהה בפועל לנתוני 2023
(כפילות/טעות בקובץ המקורי). נתוני 2022 האמיתיים, ככל שיש כאלה, יש להזין
ידנית דרך מסך "הוספת הכנסה/הוצאה".

**הערה לגבי דיוק**: תאריכי תשלומי דמי הוועד ברמת דירה בודדת לא נשמרו
בקובץ המקורי בפירוט (רק סכום מצטבר לחודש) — הם יובאו עם תאריך משוער
(ה-10 לכל חודש). ההוצאות יובאו עם התאריך המדויק כפי שמופיע בקובץ.

**איך לבצע את הייבוא**: היכנס לאפליקציה עם חשבון הוועד → **הגדרות** →
כרטיס "ייבוא נתונים היסטוריים" בתחתית → לחץ על הכפתור ואשר. זו פעולה
חד-פעמית (מומלץ שלא להריץ פעמיים, כדי לא ליצור כפילויות).

## מבנה הנתונים (Firestore)

- `settings/building-main` — שם, כתובת, דמי ועד, חתימה
- `units/{1..12}` — מספר דירה, דייר נוכחי, טלפון, PIN, היסטוריית דיירים
- `transactions/{auto}` — הכנסה/הוצאה, דירה (אם רלוונטי), קטגוריה, סכום,
  חודשים (להכנסות), תאריך, שנה
- `projects/{auto}` — פרויקטים והצעות מחיר לשיפוצים
